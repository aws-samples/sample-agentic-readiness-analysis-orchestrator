All Findings (18)

AUTH-Q1: No Machine Identity AuthenticationHIGH
Single shared API key with no per-caller identity attribution. All consumers share one identity.
Gap: No support for individual machine identity. No principal attribution in audit logs.
Recommendation: Introduce per-agent API key generation with unique principal identifiers.
AUTH-Q2: No Scoped PermissionsMEDIUM (Safety)
Binary auth model — full access or no access. No IAM policies or role definitions.
Recommendation: Implement RBAC with read-only roles and scoped API keys.
AUTH-Q3: No Action-Level AuthorizationMEDIUM (Safety)
No action-level checks. All 75+ endpoints accessible once authenticated.
Recommendation: Add action-level permission attributes on controllers.
AUTH-Q6: No Immutable Audit LoggingMEDIUM (Safety)
Local filesystem logs with no immutability. No per-caller attribution.
Recommendation: Ship logs to immutable store. Add caller identity to history records.
AUTH-Q7: No Agent Identity SuspensionMEDIUM (Safety)
Cannot suspend individual agent identities. Resetting the shared key affects all consumers.
Recommendation: Support multiple API keys with individual revocation.
STATE-Q5: No Inbound Rate LimitingMEDIUM (Safety)
No rate limiting on API endpoints. Only outbound requests are throttled.
Recommendation: Add rate limiting middleware with per-key throttling.
DATA-Q2: No Data Residency ControlsMEDIUM (Safety)
No residency enforcement. Data location depends on deployment.
Recommendation: Document residency characteristics. Keep agent-LLM traffic in-region.
DATA-Q6: No PII Redaction in LogsMEDIUM (Safety)
No log scrubbing. Stack traces in error responses. Credentials in HostConfig responses.
Recommendation: Add PII redaction middleware. Remove stack traces from production errors.
API-Q3: No Machine-Readable Error CodesMEDIUM (Quality)
Error responses have Message/Description but no error codes or retryable indicators.
Recommendation: Add errorCode and retryable fields to error model.
DISC-Q1: No Breaking Change DetectionMEDIUM (Quality)
API versioned with OpenAPI spec, but no breaking change detection in CI.
Recommendation: Add OpenAPI diff to CI pipeline.
OBS-Q1: No Distributed TracingMEDIUM (Quality)
NLog with file output. No OpenTelemetry, no correlation IDs.
Recommendation: Add OpenTelemetry instrumentation.
OBS-Q2: No AlertingMEDIUM (Quality)
No alerting on error rates or latency.
Recommendation: Configure alerting via Sentry or external monitoring.
ENG-Q1: No IaCMEDIUM (Quality)
No infrastructure as code. Deployed via platform-specific installers.
Recommendation: Create IaC for deployment environment.
ENG-Q2: No API Contract TestingMEDIUM (Quality)
Comprehensive CI/CD but no API contract or breaking change tests.
Recommendation: Add OpenAPI validation to CI.
ENG-Q3: No Automated RollbackMEDIUM (Quality)
Self-update mechanism but no automated rollback.
Recommendation: Container-based deployment with version pinning.
ENG-Q5: No Encryption at RestMEDIUM (Quality)
Unencrypted SQLite files. No TLS enforcement on PostgreSQL.
Recommendation: OS-level disk encryption. TLS for PostgreSQL.
ENG-Q4: No API Contract TestingMEDIUM (Quality)
Integration tests exist but no consumer-driven contract testing or breaking-change detection in CI.
Recommendation: Add OpenAPI spec diff validation to CI pipeline.
AUTH-Q5: Plaintext Credential StorageMEDIUM (Quality)
Credentials stored as plaintext in XML config. No secrets manager.
Recommendation: Support external secrets providers.
HITL-Q3: No Staging EnvironmentMEDIUM (Quality)
No staging environment with production-equivalent data for agent testing.
Recommendation: Create Docker-based staging with seed data.
