Executive Summary

Repository Status: 🟠 Remediation Required

Key Findings: This is a self-hosted desktop/server application (movie collection manager) with no cloud deployment infrastructure. The codebase is modern (.NET 8, React 18, TypeScript 5.7) with mature CI/CD (Azure Pipelines), but lacks all cloud-native infrastructure: no containerization, no IaC, no managed services, and no cloud security controls. The application is a tightly-coupled monolith with 2,472 C# source files in a single deployable unit.

Remediation Plan: Phase 1 focuses on containerization and foundational IaC (EKS, Aurora PostgreSQL, VPC). Phase 2 adds managed services, deployment automation, and observability. Phase 3 addresses progressive decomposition into microservices.

Recommended Actions:

  1. Containerize the application and create Dockerfile for EKS deployment
  2. Create IaC (CDK/Terraform) for VPC, EKS cluster, and Aurora PostgreSQL
  3. Integrate AWS Secrets Manager for credential management with rotation

Detailed Findings

INF-Q1: No managed compute — self-hosted applicationHIGH
Category: Infrastructure & DevOps
Finding
No managed compute infrastructure exists. The application is distributed as self-contained .NET 8 executables and runs as a self-hosted service. No ECS, EKS, Lambda, or Fargate resources defined.
Recommendation
Containerize the application and deploy to EKS (preferred). Create a Dockerfile using the .NET 8 runtime base image.
INF-Q2: All databases self-managedHIGH
Category: Infrastructure & DevOps
Finding
SQLite embedded database and optional self-managed PostgreSQL. No managed database service (Aurora, RDS).
Recommendation
Migrate to Aurora PostgreSQL. The application already supports PostgreSQL natively via FluentMigrator.
INF-Q5: No cloud network security definedHIGH
Category: Infrastructure & DevOps
Finding
No VPC, subnets, security groups, or network segmentation. Application binds directly to port 7878.
Recommendation
Define VPC with private subnets. Use security groups for least-privilege access. Deploy API Gateway as public entry point.
