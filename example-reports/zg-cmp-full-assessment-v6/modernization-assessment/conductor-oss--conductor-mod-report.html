Executive Summary

Repository Status: 🟠 Remediation Required

This workflow orchestration engine (originally from Netflix) demonstrates strong application-level architecture with modern Java 21, Spring Boot 3.3.11, and extensive AI integration. However, it has significant infrastructure, security, and operational gaps that prevent production-ready cloud deployment. No Infrastructure as Code exists, no authentication protects the API, and no observability beyond basic metrics is configured.

Key Findings

  • 7 High severity findings primarily in Infrastructure and Security categories
  • No IaC, no managed compute, no managed databases — entirely Docker Compose based
  • Zero authentication on all API endpoints — critical security gap
  • No distributed tracing despite being a distributed workflow orchestrator

Remediation Plan

Three modernization pathways triggered: Move to Cloud Native (High priority), Move to Managed Databases (High), Move to Modern DevOps (High). Start with IaC foundation and security baseline before architectural decomposition.

Recommended Actions

  1. Create Terraform/CDK for EKS, Aurora, VPC infrastructure
  2. Add API Gateway with Cognito authentication
  3. Migrate databases to Aurora PostgreSQL and ElastiCache

Detailed Findings

INF-Q1: No managed compute infrastructureHIGH
Category: Infrastructure & DevOps

Finding

No managed compute infrastructure is defined. Only Docker Compose for local development and Docker images published to Docker Hub.

Recommendation

Deploy to Amazon EKS with managed node groups. Create Kubernetes Deployment manifests for the conductor-server container image.

INF-Q2: All databases self-managedHIGH
Category: Infrastructure & DevOps

Finding

All databases self-managed via Docker Compose: PostgreSQL 16, MySQL, Redis 6.2.3, Elasticsearch 7.17.11.

Recommendation

Migrate to Aurora PostgreSQL, ElastiCache for Redis, and Amazon OpenSearch Service.

INF-Q5: No network security configurationHIGH
Category: Infrastructure & DevOps

Finding

No VPC, security groups, subnets, or NACLs defined. Docker Compose exposes ports directly.

Recommendation

Define VPC with private subnets. Use security groups with least-privilege rules. Place Conductor behind API Gateway or ALB.

INF-Q10: Zero Infrastructure as CodeHIGH
Category: Infrastructure & DevOps

Finding

No Terraform, CloudFormation, CDK, Helm charts, or Kustomize files. All infrastructure is ClickOps.

Recommendation

Create Terraform modules for EKS cluster, Aurora PostgreSQL, ElastiCache, VPC networking, API Gateway, IAM roles, CloudWatch.

SEC-Q1: No audit loggingHIGH
Category: Security Baseline

Finding

No CloudTrail or audit logging configured. Only Log4j2 console appender for application logs.

Recommendation

Enable CloudTrail. Configure application-level audit logging. Use S3 with Object Lock for immutable archive.

SEC-Q2: No encryption at restHIGH
Category: Security Baseline

Finding

No encryption configured. Elasticsearch security explicitly disabled (xpack.security.enabled=false).

Recommendation

Enable Aurora encryption with KMS, ElastiCache encryption, OpenSearch encryption. Use SSE-KMS for S3.

OPS-Q2: No SLO definitionsHIGH
Category: Operations & Observability

Finding

Prometheus metrics with percentile tracking but no formal SLO definitions or error budgets.

Recommendation

Define SLOs for workflow execution latency, task scheduling latency, and API p99 response time.

SEC-Q3: No API authenticationMEDIUM
Category: Security Baseline

Finding

No Spring Security, no OAuth2/JWT, no API keys. Server is completely open to any caller.

Recommendation

Deploy API Gateway with Cognito authorizer. Add Spring Security with OAuth2 resource server.

APP-Q2: Modular monolith, single deploymentMEDIUM
Category: Application Architecture

Finding

48 Gradle subprojects compile into a single Spring Boot JAR. Shared database schemas between modules.

Recommendation

Begin Strangler Fig extraction starting with the AI module (clearest bounded context).

SEC-Q5: Secrets in env vars without rotationMEDIUM
Category: Security Baseline

Finding

All secrets via environment variables. No plaintext in source but no Secrets Manager or Vault integration.

Recommendation

Integrate with AWS Secrets Manager. Configure automatic rotation. Use EKS Secrets Store CSI Driver.

OPS-Q1: No distributed tracingMEDIUM
Category: Operations & Observability

Finding

No OpenTelemetry SDK, X-Ray, Jaeger, or Zipkin. Cannot trace request flows across workflow execution.

Recommendation

Add OpenTelemetry Java Agent. Configure OTLP exporter to X-Ray via ADOT Collector.

🟠 Modernization Recommendations

  1. Establish Infrastructure Foundation: Create Terraform/CDK for EKS, Aurora PostgreSQL, ElastiCache, VPC — this unblocks all other modernization work.
  2. Secure the API Surface: Deploy API Gateway with Cognito authentication and request validation. The completely open server is the most critical security gap.
  3. Migrate to Managed Databases: Move PostgreSQL to Aurora, Redis to ElastiCache, Elasticsearch to OpenSearch Service. The pluggable persistence architecture makes this a configuration change.
