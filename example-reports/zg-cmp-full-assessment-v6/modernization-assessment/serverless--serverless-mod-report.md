## Quick Agent Wins

### RAG-based Knowledge Agent

- **Prerequisite:** Documentation exists — extensive `docs/` directory with markdown files covering all CLI features, AWS events, and configuration options. README.md provides comprehensive overview.
- **What it enables:** A knowledge agent that indexes the framework documentation and provides natural language assistance for users configuring serverless.yml files, troubleshooting deployments, and understanding framework features.
- **Additional steps:** Documentation is already structured markdown — can be directly indexed. No OpenAPI spec exists for the MCP server but the tool definitions in `packages/mcp/src/` provide structured interfaces.
- **Effort:** Low

### DevOps Agent

- **Prerequisite:** CI/CD pipeline exists (INF-Q11 equivalent is 3+ — full GitHub Actions CI/CD with test, build, deploy, and release stages).
- **What it enables:** An agent that monitors CI status, triggers releases, checks cross-platform test results, and manages canary→stable promotion.
- **Additional steps:** GitHub Actions API provides the automation surface. Agent would need GitHub token with appropriate permissions.
- **Effort:** Medium

### Observability Agent

- **Prerequisite:** Structured logging and tracing in place (OPS-Q1 = 3) — Axiom integration provides structured observability data.
- **What it enables:** An agent that queries Axiom logs, identifies deployment failure patterns, and surfaces common user errors.
- **Additional steps:** Axiom API access needed; agent would query structured logs for pattern detection.
- **Effort:** Medium
