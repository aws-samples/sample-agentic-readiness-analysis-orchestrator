INF-Q1: No Managed Compute InfrastructureHIGH
Category: Infrastructure & DevOps
Finding
No managed compute (ECS/EKS/Lambda/Fargate) defined. No IaC for any compute resources. Application is packaged for self-managed deployment via Sonatype Nexus.
Recommendation
Containerize the application and deploy on Amazon EKS. Create a production Dockerfile, define Kubernetes manifests with StatefulSets for journal-dependent components.
INF-Q2: All Databases Self-ManagedHIGH
Category: Infrastructure & DevOps
Finding
MongoDB via localhost connection string. OpenSearch managed by custom data-node module. Docker-compose defines self-hosted MongoDB 7.0 and OpenSearch 2.10.
Recommendation
Migrate MongoDB to Amazon DocumentDB and OpenSearch to Amazon OpenSearch Service for automated backups, Multi-AZ failover, and managed patching.
INF-Q5: No Network Security ConfigurationHIGH
Category: Infrastructure & DevOps
Finding
No VPC, subnet, security group, or NACL definitions. No IaC defining network topology. TLS disabled by default.
Recommendation
Define VPC infrastructure with private subnets. Use security groups with least-privilege rules. Enable TLS by default. Consider VPC endpoints for AWS service access.
INF-Q10: Zero Infrastructure as CodeHIGH
Category: Infrastructure & DevOps
Finding
No Terraform, CloudFormation, CDK, Helm charts, or Kustomize files found. All infrastructure created and managed externally.

SEC-Q1: No CloudTrail Audit LoggingHIGH
Category: Security Baseline
Finding
No CloudTrail or equivalent infrastructure audit logging configured. OpenSearch security audit covers only the OpenSearch layer.

SEC-Q2: No Encryption at RestHIGH
Category: Security Baseline
Finding
No encryption at rest configuration. No KMS keys. Docker-compose runs databases without encryption. Only application-level EncryptedValue for stored credentials.
Recommendation
Enable encryption at rest on DocumentDB (KMS), OpenSearch Service (KMS), and EBS volumes. Define customer-managed KMS keys with rotation policy.
