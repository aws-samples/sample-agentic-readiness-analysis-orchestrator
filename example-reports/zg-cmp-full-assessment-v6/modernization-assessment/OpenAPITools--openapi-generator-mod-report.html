Repository Status: 🟠 Remediation Required

This repo has 3 High findings, 18 Medium findings, 4 Low findings. Rule matched: 2-11 High → Remediation Required.

Key Findings: The OpenAPI Generator is a well-structured, modular Java code-generation toolkit with extensive CI/CD automation (105 GitHub Actions workflows). However, it lacks cloud deployment infrastructure entirely — no IaC, no managed compute, no network security, no observability, and no authentication on its online service. The application architecture is mature (Score 4 on decomposition), but the infrastructure and operations layers are almost entirely absent.

Remediation Plan: Phase 1 focuses on foundational infrastructure (IaC, VPC, managed compute, audit logging). Phase 2 addresses security hardening (authentication, identity, vulnerability scanning) and observability (tracing, metrics, alerting). Phase 3 covers operational maturity (incident response, runbooks).

Recommended Actions:

  1. Define IaC for deploying the online service to EKS with proper VPC/networking
  2. Add API Gateway with authentication in front of the online service
  3. Instrument observability (OpenTelemetry, CloudWatch alarms, business metrics)

Detailed Findings

INF-Q1: No managed compute orchestration HIGH

Category: Infrastructure & DevOps
FINDING

All compute is defined only as Docker images without any managed container orchestration or serverless deployment. Images are published to Docker Hub but no ECS, EKS, Fargate, Lambda, or App Runner resources are defined.

RECOMMENDATION

Deploy the online service to EKS (per preferences) or ECS Fargate with managed health checks, auto-scaling, and service discovery. Define the deployment as IaC.

INF-Q5: No network security configuration HIGH

Category: Infrastructure & DevOps
FINDING

No VPC, subnet, security group, NACL, or network segmentation configuration found. The online service exposes port 8080 via Docker with no network isolation defined.

RECOMMENDATION

Define VPC with private subnets for the online service. Deploy behind an ALB or API Gateway in public subnets with the application in private subnets.

SEC-Q1: No audit logging HIGH

Category: Security Baseline
FINDING

No CloudTrail, audit logging, or structured request logging. SLF4J provides basic console logging only.

RECOMMENDATION

Enable CloudTrail. Add structured request logging (JSON) with correlation IDs. Ship logs to CloudWatch Logs with retention policy.

Category: Infrastructure & DevOps
FINDING

No auto-scaling configuration found. The online service runs as a single Docker container with no scaling mechanisms.

RECOMMENDATION
