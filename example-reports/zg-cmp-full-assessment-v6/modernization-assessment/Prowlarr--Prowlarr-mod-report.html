HIGH INF-Q2: Self-Managed Databases

Category: Infrastructure & DevOps
Finding

All databases are self-managed. SQLite is embedded default; PostgreSQL requires user-provisioned instance. No managed database services referenced.

Recommendation

Migrate PostgreSQL workloads to Amazon Aurora PostgreSQL. Application already has full PostgreSQL support via Npgsql driver.

HIGH INF-Q5: No Network Security Infrastructure

Category: Infrastructure & DevOps
Finding

No VPC, security groups, NACLs, or network segmentation defined. Application exposes port 9696 directly with permissive CORS.

Recommendation

Provision VPC with private subnets for application and database. Use security groups with least-privilege rules. Place ALB or API Gateway in public subnet.

HIGH APP-Q2: Tightly-Coupled Monolith

Category: Application Architecture
Finding

Single deployable binary with all functionality bundled. Single-instance enforcement. All modules share database via BasicRepository with cross-module table access.

Recommendation

Begin Strangler Fig extraction: Notifications service first (lowest coupling), then History, then Indexer Search.

HIGH SEC-Q1: No Cloud Audit Logging

Category: Security Baseline
Finding

No CloudTrail or equivalent. Application-level NLog with Sentry error tracking. No immutable audit trail.

Recommendation

Enable CloudTrail with log file validation and S3 Object Lock. Ship audit logs to CloudWatch Logs.

HIGH SEC-Q2: No Encryption at Rest

Category: Security Baseline
Finding

SQLite database files, Config.xml with credentials, and backup ZIPs all stored unencrypted on filesystem.

Recommendation

Enable Aurora encryption with customer-managed KMS key. Store backups in S3 with SSE-KMS. Use Secrets Manager for credentials.
