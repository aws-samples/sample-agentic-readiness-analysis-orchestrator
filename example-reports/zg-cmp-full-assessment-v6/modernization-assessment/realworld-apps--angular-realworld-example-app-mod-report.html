Executive Summary

Repository Status: 🟠 Remediation Required

Key Findings: This Angular frontend SPA has modern language and framework choices (TypeScript 5.9, Angular 21) but lacks infrastructure automation, security baseline controls, and operational observability. The application is deployed as a static monolith to Netlify with no IaC, no audit logging, no centralized identity, and no monitoring.

Remediation Plan: Prioritize IaC definition (S3+CloudFront hosting via CDK), CI/CD deployment automation, and dependency security scanning. These foundational DevOps improvements enable all subsequent modernization.

Recommended Actions:

  1. Define hosting infrastructure in CDK/Terraform (S3 + CloudFront + Route 53)
  2. Add Dependabot and npm audit to CI pipeline
  3. Integrate Amazon Cognito for identity management

Detailed Findings

INF-Q10: No Infrastructure as CodeHIGH
Category: Infrastructure & DevOps
Finding
No infrastructure-as-code exists in this repository. Hosting infrastructure (Netlify) is configured entirely outside the codebase with no Terraform, CDK, CloudFormation, or Helm definitions.
Recommendation
Define hosting infrastructure in Terraform or CDK: S3 bucket for static assets, CloudFront distribution with OAC, Route 53 DNS records, ACM certificate for TLS.
APP-Q2: Single Deployable Monolith SPAHIGH
Category: Application Architecture
Finding
The application is a single deployable unit — one Angular build produces one bundle. All features share global state, a single router, and produce a single build artifact with no independent deployment capability.
Recommendation
For a reference/demo app of this size, full decomposition is not warranted. If independent deployment is desired: adopt Angular Module Federation for per-feature-module builds.
SEC-Q1: No Audit LoggingHIGH
Category: Security Baseline
Finding
No CloudTrail or equivalent audit logging is configured. No infrastructure exists in this repository to enable audit logging.
Recommendation
When migrating to AWS hosting: enable CloudTrail, configure CloudFront access logging to S3, and add CloudWatch RUM for client-side event logging.
SEC-Q4: No Centralized Identity ProviderHIGH
Category: Security Baseline
Finding
The application manages its own authentication entirely with no external IdP integration. Login/register flows send credentials directly to the backend API.
Recommendation
Integrate with Amazon Cognito as identity provider. Use Cognito Hosted UI for OAuth2/OIDC with SSO, social login, and MFA.
SEC-Q6: No Compute Hardening or Vulnerability ScanningHIGH
Category: Security Baseline
Finding
No compute hardening or patching strategy exists. No vulnerability scanning (Inspector, Snyk) configured for the deployment environment.
Recommendation
Focus on dependency vulnerability scanning in CI (npm audit, Dependabot). If containerized, use ECR image scanning and Bottlerocket base images.
OPS-Q1: No Distributed TracingHIGH
Category: Operations & Observability
Finding
No distributed tracing instrumented. No OpenTelemetry SDK, no X-Ray, no trace ID propagation.
Recommendation
Add OpenTelemetry browser SDK or AWS CloudWatch RUM for frontend performance monitoring and trace context propagation.
OPS-Q3: No Business MetricsHIGH
Category: Operations & Observability
Finding
No custom business metrics published. No analytics, no CloudWatch custom metrics, no user engagement tracking.
Recommendation
Add CloudWatch RUM for real user monitoring. Instrument key user journeys with custom metrics.
OPS-Q4: No Anomaly Detection or AlertingHIGH
Category: Operations & Observability
Finding
No anomaly detection or alerting configured. No CloudWatch alarms, no error rate monitoring, no latency alerts.
Recommendation
Configure CloudWatch alarms on CloudFront error rates. Add CloudWatch Synthetics for critical user journey canaries.
INF-Q11: Partial CI/CD — Build Automated but Deployment ManualMEDIUM
Category: Infrastructure & DevOps
Finding
GitHub Actions workflows exist for build, lint, and E2E testing. However, deployment to production is not controlled by this pipeline.
Recommendation
Extend GitHub Actions to deploy build artifacts to AWS (S3 upload + CloudFront invalidation). Add environment promotion with approval gates.
APP-Q6: Hard-coded API EndpointMEDIUM
Category: Application Architecture
Finding
External API endpoint hard-coded in api interceptor. Cannot switch environments without code changes.
Recommendation
Externalize the API base URL using Angular environment files or runtime configuration injection.
SEC-Q3: Client-side JWT Without OAuth2/OIDCMEDIUM
Category: Security Baseline
Finding
JWT token stored in localStorage (XSS-vulnerable). No OAuth2/OIDC flow, no API Gateway authorizer.
Recommendation
Integrate with Amazon Cognito for OAuth2/OIDC with PKCE. Use httpOnly cookies for token management.
SEC-Q5: Secrets in GitHub Actions Without RotationMEDIUM
Category: Security Baseline
Finding
CI/CD secrets in GitHub Actions without rotation. Static AWS access keys without OIDC federation.
Recommendation
Use OIDC federation for GitHub Actions to AWS (eliminates static access keys). Use Secrets Manager with rotation.
SEC-Q7: No SAST or Dependency Scanning in CIMEDIUM
Category: Security Baseline
Finding
Security E2E tests exist but no SAST tool, no Dependabot, no npm audit in CI pipeline.
Recommendation
Add Dependabot for dependency alerts. Add npm audit --audit-level=critical as blocking CI step. Consider Semgrep for SAST.
APP-Q5: API Versioning via Spec OnlyLOW
Category: Application Architecture
Finding
API has OpenAPI 3.1.0 spec (v2.0.0) but no explicit version segment in URL path.
Recommendation
Pin OpenAPI spec version and add contract testing to detect breaking changes from the external API.
