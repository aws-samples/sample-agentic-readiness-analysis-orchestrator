Findings (3 Medium)

Medium DISC-Q1: No automated API contract breaking-change detection

Category: Discovery & Documentation | Priority: P2 | Effort: Medium | Phase: 2

No automated breaking-change detection in CI (no japicmp, revapi, buf breaking, or consumer-driven contract tests). Public API surface changes could break downstream Flink applications without early detection.

Recommendation: Add japicmp-maven-plugin or revapi-maven-plugin to CI for binary compatibility checks.

Medium OBS-Q1: No structured logging or distributed tracing hooks

Category: Observability | Priority: P2 | Effort: Medium | Phase: 2

SLF4J with plain-text PatternLayout. No structured JSON logging, no MDC/NDC context enrichment, no OpenTelemetry/X-Ray instrumentation hooks.

Recommendation: Add MDC context propagation for trace IDs in key connector operations.

Medium OBS-Q2: No recommended alerting thresholds documented

Category: Observability | Priority: P2 | Effort: Low | Phase: 2

Library emits Flink metrics but provides no recommended alerting thresholds documentation for consuming applications.

Recommendation: Add Monitoring & Alerting documentation section with recommended metric thresholds.
