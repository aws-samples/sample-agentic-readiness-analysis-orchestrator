Findings (8)

Medium AUTH-Q2: No scoped permissions — all endpoints publicly accessible

Gap: No scoped permission model exists. Any caller has identical unrestricted access to all endpoints.

Recommendation: Deploy behind AWS API Gateway with resource-level IAM policies.

Safety Impact: Yes

Priority: P1 | Effort: Medium | Phase: 1

Medium AUTH-Q3: No action-level authorization — GET and POST equally accessible

Gap: No action-level authorization. An agent intended for read-only access could invoke POST endpoints.

Recommendation: Add API Gateway method-level authorization or Spring Security for GET/POST differentiation.

Safety Impact: Yes

Priority: P1 | Effort: Medium | Phase: 1

Medium STATE-Q5: No rate limiting — CPU-intensive endpoints unprotected

Gap: No rate limiting. Runaway agent could overwhelm service via repeated POST calls.

Recommendation: Deploy behind API Gateway with usage plans and throttle settings.

Safety Impact: Yes

Priority: P1 | Effort: Medium | Phase: 1

Medium API-Q2: No static machine-readable API specification committed

Gap: No static OpenAPI spec. Agent tool generation requires a spec file at build time.

Recommendation: Export Springfox-generated spec to a static openapi.yaml committed to the repo.

Safety Impact: No

Priority: P2 | Effort: Low | Phase: 2

Medium API-Q3: No structured error responses with retryable classification

Gap: No machine-readable error category (retryable vs terminal).

Recommendation: Add @ControllerAdvice with structured error response format.

Safety Impact: No

Priority: P2 | Effort: Low | Phase: 2

Medium OBS-Q1: No distributed tracing or structured logging

Gap: Cannot trace agent-initiated requests. No structured logs.

Recommendation: Add Spring Cloud Sleuth and JSON logging configuration.

Safety Impact: No

Priority: P2 | Effort: Low | Phase: 2

Medium OBS-Q2: No alerting on error rates or latency

Gap: Service degradation undetected. No actuator, no alarms.

Recommendation: Add spring-boot-starter-actuator and configure alerting.

Safety Impact: No

Priority: P2 | Effort: Low | Phase: 2

Medium DISC-Q1: No API versioning or breaking change detection

Gap: No versioning, no breaking change detection. Agent tool bindings may break silently.

Recommendation: Add URL-based versioning and OpenAPI diff tooling in CI.

Safety Impact: No

Priority: P2 | Effort: Medium | Phase: 2
