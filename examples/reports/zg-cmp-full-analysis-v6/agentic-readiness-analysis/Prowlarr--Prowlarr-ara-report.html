Findings

BLOCKERAUTH-Q1: No machine identity authentication
Finding
Single shared API key for all consumers. No per-agent identity or principal attribution.
Gap
Cannot distinguish which agent made a call. No per-agent credentials.
Recommendation
Implement multiple API key support with principal labels and attribution in audit logs.
RISK-SAFETYAUTH-Q2: No scoped permissions
Finding
Binary authentication — valid API key grants full access to all operations.
Recommendation
Implement RBAC with at least read-only and admin roles.
RISK-SAFETYAUTH-Q3: No action-level authorization
Finding
No distinction between read and write operations once authenticated.
Recommendation
Add authorization policy distinguishing read from write operations.
RISK-SAFETYAUTH-Q6: No immutable audit logging
Finding
Auth events logged locally. No immutable audit trail for API operations.
Recommendation
Add audit middleware and ship to immutable log store.
RISK-SAFETYAUTH-Q7: No per-agent identity suspension
Finding
Single API key cannot be selectively revoked per agent.
Recommendation
Implement multi-key support with per-key disable capability.
RISK-SAFETYSTATE-Q5: No server-side rate limiting
Finding
No rate limiting on inbound API calls. Only outbound rate limiting exists.
Recommendation
Add ASP.NET Core rate limiting middleware.
RISK-SAFETYDATA-Q1: Sensitive credentials exposed in API
Finding
HostConfigResource returns ApiKey, SslCertPassword, ProxyPassword directly in API responses.
Recommendation
Create separate DTOs excluding credentials. Add [JsonIgnore] to sensitive fields.
RISK-SAFETYDATA-Q6: Reactive PII redaction in logs
Finding
40+ regex patterns for log cleansing. Reactive pattern-based rather than proactive allowlisting.
Recommendation
Migrate to allowlist-based logging model for sensitive contexts.
RISK-QUALITYAPI-Q2: OpenAPI spec lacks CI validation
Finding
OpenAPI spec exists but no automated drift detection in CI.
Recommendation
Add CI step to validate spec-code synchronization.
RISK-QUALITYAPI-Q3: Error responses lack retryable classification
Finding
Structured error bodies without retryable indicator or machine-readable error codes.
Recommendation
Add error code and retryable boolean to response envelope.
RISK-QUALITYDISC-Q1: No breaking change detection
Finding
API versioned at URL level. No contract testing or breaking change detection in CI.
Recommendation
Add OpenAPI diff tooling and contract tests.
RISK-QUALITYOBS-Q1: No distributed tracing
Finding
No OpenTelemetry, no correlation IDs, no structured logging by default.
Recommendation
Add OpenTelemetry and correlation ID middleware.
RISK-QUALITYOBS-Q2: No alerting configured
Finding
No alerting for API error rates or latency.
Recommendation
Add metrics export with alert thresholds.
RISK-QUALITYENG-Q1: No IaC in repository
Finding
No Terraform, CloudFormation, CDK, or Helm charts. Self-contained archive distribution.
Recommendation
Create external IaC for deployment infrastructure.
RISK-QUALITYENG-Q2: No API contract testing
Finding
Comprehensive CI/CD but no contract tests for agent-facing APIs.
Recommendation
Add Pact or OpenAPI diff testing to pipeline.
RISK-QUALITYENG-Q3: Manual rollback only
Finding
File-system backup/restore. No automated platform-level rollback.
Recommendation
Containerize with orchestration supporting automated rollback.
RISK-QUALITYENG-Q5: No encryption at rest
Finding
SQLite plain files. Credentials accessible with filesystem access.
Recommendation
Deploy on encrypted volumes or use SQLCipher.
RISK-QUALITYDATA-Q3: Offset-only pagination
Finding
Page-based offset pagination. No cursor-based pagination or field selection.
Recommendation
Add max page size enforcement and cursor-based pagination.
