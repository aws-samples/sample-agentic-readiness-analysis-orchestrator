BLOCKER (1)

AUTH-Q1: Machine Identity AuthenticationBLOCKER
SIMPLE auth is trust-based with no credential verification. No OAuth2, no API key management, no mTLS. S3 REST auth disabled by default (PassAllAuthenticator).
Gap: No machine identity authentication suitable for agent principals.

RISK-SAFETY (6)

AUTH-Q2: Scoped PermissionsRISK-SAFETY
POSIX-style permissions + ACLs at path level only. No operation-level permission scoping for agent identities.
Gap: Cannot scope agent permissions per API operation.
AUTH-Q3: Action-Level AuthorizationRISK-SAFETY
Authorization is rwx at path level. Cannot differentiate create from delete within write permission.
Gap: No action-level authorization for API operations.
AUTH-Q6: Immutable Audit LoggingRISK-SAFETY
Audit logging exists but disabled by default and stored as rolling files without immutability.
Gap: Not immutable, not tamper-evident, disabled by default.
AUTH-Q7: Agent Identity SuspensionRISK-SAFETY
No mechanism to suspend individual agent identities without service restart. Channel cleanup every 3 days.
Gap: No real-time identity revocation mechanism.
STATE-Q4: Circuit BreakersRISK-SAFETY
Retry policies exist but no circuit breakers for external UFS calls. Failing UFS can cascade through master.
Gap: No circuit breaker pattern for external dependencies.
STATE-Q5: Rate LimitingRISK-SAFETY
Server-wide rate limiting present but disabled by default. No per-identity limits.
Gap: No per-client/per-identity request rate limiting active by default.

RISK-QUALITY (12)

API-Q2: Machine-Readable SpecRISK-QUALITY
gRPC protobuf definitions exist. No OpenAPI spec committed for REST proxy API.
Gap: No committed OpenAPI/Swagger spec for REST surface.
API-Q3: Structured ErrorsRISK-QUALITY
S3 API has structured errors. Paths REST API lacks structured error responses.
Gap: Non-S3 REST API lacks differentiated error responses.
HITL-Q3: Sandbox/StagingRISK-QUALITY
Docker Compose and minicluster available. No staging with production-equivalent data.
Gap: No documented staging/sandbox environment.
DATA-Q4: System of RecordRISK-QUALITY
Alluxio is explicitly a caching layer. No formal SoR documentation for agents.
Gap: No formal system-of-record designation documentation.
DATA-Q5: Temporal MetadataRISK-QUALITY
Timestamps exist but no freshness signaling in API responses.
Gap: Agents cannot determine if data is fresh or stale from responses.
DISC-Q1: Schema VersioningRISK-QUALITY
Protobuf definitions unversioned. No breaking-change detection in CI.
Gap: No API versioning or contract stability checks.
OBS-Q1: Distributed TracingRISK-QUALITY
OpenTelemetry auto-instrumentation present. Application logs not structured JSON, no trace IDs in logs.
Gap: Logs not structured. No trace ID correlation in application logs.
OBS-Q2: AlertingRISK-QUALITY
Rich metrics published to Prometheus. No alerting rules defined.
Gap: No alerting thresholds configured.
ENG-Q1: Infrastructure GovernanceRISK-QUALITY
Helm charts define deployment. No drift detection, no comprehensive IaC for integration surface.
Gap: No drift detection, no IaC beyond Helm.
ENG-Q2: API Contract TestingRISK-QUALITY
Unit/integration tests present. No API contract tests or proto breaking-change detection.
Gap: API-breaking changes not caught before merge.
ENG-Q3: Rollback CapabilityRISK-QUALITY
No CD pipeline. Manual Helm rollback only. No automated triggers.
Gap: No automated rollback capability.
ENG-Q5: Encryption at RestRISK-QUALITY
No service-level encryption. Cached data on workers stored unencrypted. Delegated to UFS.
