Findings

HIGH AUTH-Q1: No machine identity authentication with principal attribution Authentication & Authorization

Gap: No machine identity authentication mechanism exists that can distinguish which agent made a call.

Recommendation: Create dedicated IAM-style accounts per agent in the Vault backend with distinct access keys. Enable server access logging by default.

Evidence: lib/auth/vault.js, conf/authdata.json

MEDIUM AUTH-Q2: Scoped permissions incomplete — IAM integration pending Authentication & Authorization

Gap: IAM-level scoped permissions may be incomplete. Service user bypass grants full access.

Recommendation: Complete IAM integration. Never classify agents as service users.

MEDIUM AUTH-Q3: Action-level authorization depends on incomplete IAM Authentication & Authorization

Gap: IAM action-level enforcement may be incomplete.

Recommendation: Apply explicit bucket policies to agent-accessible buckets.

MEDIUM AUTH-Q6: Audit logging file-based, mutable, disabled by default Authentication & Authorization

Gap: Audit logs are not immutable. Disabled by default.

Recommendation: Enable by default. Forward to immutable store.

MEDIUM AUTH-Q7: No immediate agent identity suspension mechanism Authentication & Authorization

Gap: No hot-reloadable identity suspension.

Recommendation: Implement access key deny-list with hot-reload.

MEDIUM STATE-Q4: No circuit breakers for external dependencies State Management

Gap: No circuit breakers, retry logic, or configurable timeouts.

Recommendation: Implement circuit breaker patterns for MongoDB, Redis, Vault.

MEDIUM STATE-Q5: Rate limiting per-bucket only, disabled, fails open State Management

Gap: No per-agent rate limiting. Disabled by default. Fails open.

Recommendation: Enable by default. Add per-identity limiting.

MEDIUM DATA-Q1: Service user bypass — undifferentiated data access Data Accessibility

Gap: Service user bypass provides full access with no granularity.

Recommendation: Implement role-based access tiers. Never grant agents service user status.

MEDIUM DATA-Q2: Data residency configurable but no egress enforcement Data Accessibility

Gap: No controls prevent data egress via API once read.

Recommendation: Add residency metadata for orchestration-layer enforcement.

MEDIUM DATA-Q6: No PII redaction in server access logs Data Accessibility

Gap: IP addresses, access keys, object keys logged unredacted.

Recommendation: Add configurable PII redaction filter.

MEDIUM API-Q2: No machine-readable API specification API Surface

Gap: No OpenAPI/Smithy spec. API follows S3 protocol implicitly.

Recommendation: Reference AWS S3 API model or generate OpenAPI spec.

MEDIUM API-Q3: No explicit retryable indicator in errors API Surface

Gap: No retryable boolean. Agents infer from HTTP status codes.

Recommendation: Document retry semantics per error code.

MEDIUM OBS-Q1: No distributed tracing Observability

Gap: Cannot trace requests through backend dependencies.

Recommendation: Add OpenTelemetry instrumentation.

MEDIUM OBS-Q2: Alerting requires external AlertManager Observability

Gap: No alerting infrastructure deployed. No auth anomaly alerts.

Recommendation: Deploy AlertManager. Add auth alerts.

MEDIUM DISC-Q1: No API contract testing or breaking change detection Discovery & Documentation

Gap: No schema diffing or contract tests in CI.

Recommendation: Add OpenAPI diff checking in CI.

MEDIUM ENG-Q1: No Infrastructure as Code Engineering Maturity

Gap: No IaC for deployment infrastructure.

Recommendation: Create IaC for production deployment.

MEDIUM ENG-Q2: CI lacks API contract testing Engineering Maturity

Gap: No API contract testing in pipeline.

Recommendation: Add S3 API compatibility tests in CI.

MEDIUM ENG-Q3: No automated rollback Engineering Maturity

Gap: Manual Docker image revert only.

Recommendation: Implement health-check-based rollback.

MEDIUM HITL-Q3: No production-equivalent staging Human-in-the-Loop

Gap: No staging with realistic data shape.

Recommendation: Define staging configuration with seed data.
