HIGH AUTH-Q1: No Machine Identity Authentication

Gap: Single shared API key with no principal attribution. Cannot distinguish which client made a specific API call.

Recommendation: Implement per-client API key issuance with principal field for audit attribution.

Evidence: src/Radarr.Http/Authentication/ApiKeyAuthenticationHandler.cs

MEDIUM AUTH-Q2: No Scoped Permissions

Gap: Single API key grants full access to all 164 endpoints. No least-privilege enforcement.

Recommendation: Introduce role-based API keys mapping to allowed endpoint sets.

MEDIUM AUTH-Q3: No Action-Level Authorization

Gap: Cannot restrict agents to specific actions within a resource type.

Recommendation: Add per-method authorization policies on controllers.

MEDIUM AUTH-Q6: No Immutable Audit Logging

Gap: No principal attribution in logs. Log files are mutable.

Recommendation: Add principal identity to HTTP logging. Forward to immutable external store.

MEDIUM AUTH-Q7: No Agent Identity Suspension

Gap: Cannot isolate misbehaving agents without full API lockout.

Recommendation: Implement per-client keys with active/suspended status.

MEDIUM STATE-Q5: No Inbound API Rate Limiting

Gap: No protection against agent traffic storms.

Recommendation: Add ASP.NET Core rate limiting middleware with per-client policies.

MEDIUM DATA-Q1: Sensitive Credentials in API Responses

Gap: HostConfig endpoint returns API key, password hash, proxy credentials.

Recommendation: Add [JsonIgnore] to sensitive fields. Create separate credential endpoints.

MEDIUM DATA-Q6: Potential PII in Trace Logs

Gap: Trace-level body logging could capture user data. Regex scrubbing may miss patterns.

Recommendation: Audit trace-level logging. Consider structured log field allowlist.

MEDIUM API-Q2: OpenAPI Spec Lacks Enrichment

Gap: Response schemas lack descriptions and examples.

Recommendation: Enrich OpenAPI spec with examples for key endpoints.

MEDIUM API-Q3: Error Responses Lack Machine Codes

Gap: No error code enum or retryable flag.

Recommendation: Add errorCode enum and retryable boolean to ErrorModel.

MEDIUM DATA-Q3: Primary Endpoint Lacks Pagination

Gap: GET /api/v3/movie returns unbounded results.

Recommendation: Add pagination to primary movie endpoint.

MEDIUM DISC-Q1: No Breaking-Change Detection

Gap: No API contract testing in CI.

Recommendation: Add OpenAPI spec diff as CI step.

MEDIUM OBS-Q1: No Distributed Tracing

Gap: No OpenTelemetry or correlation IDs.

Recommendation: Add OpenTelemetry SDK with ASP.NET Core instrumentation.

MEDIUM OBS-Q2: No Alerting Thresholds

Gap: No error rate or latency alerting.

Recommendation: Expose Prometheus metrics. Configure alerting.

MEDIUM ENG-Q1: No Infrastructure as Code

Gap: No IaC for agent-accessible deployment.

Recommendation: Create reference IaC templates.

MEDIUM ENG-Q2: No API Contract Testing

Gap: API changes not validated against contract.

Recommendation: Add OpenAPI spec diff in CI pipeline.

MEDIUM ENG-Q3: No Automated Rollback

Gap: No fast rollback for broken releases.

Recommendation: Add down-migration support. Document rollback procedures.
