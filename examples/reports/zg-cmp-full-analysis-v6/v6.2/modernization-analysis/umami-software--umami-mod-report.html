Executive Summary

Repository Status: 🟠Remediation Required — Significant Modernization Required. 6 High findings, 26 Medium, 1 Low across 33 emitted findings; V5 overall score 1.79 / 4.0 (Needs Work). V5/V6 classification consistency: consistent.

Key Findings: Umami ships as a self-hosted OSS distribution with Dockerfile + docker-compose. No IaC, no managed compute target, and no managed database path are shipped with the repo. Security posture leans on env-var secrets and a JWT-over-APP_SECRET auth model. Operationally, Cypress E2E exists but is not wired into CI.

Remediation Plan: Adopt CDK for a reference EKS + Aurora deployment (preferences.prefer). Migrate the optional Kafka path to EventBridge or MSK Serverless. Wire Cypress into CI. Introduce API versioning. Store secrets in AWS Secrets Manager.

Recommended Actions: Start with the CDK IaC reference, then layer managed Aurora + ElastiCache, container scanning in CI/CD, OpenTelemetry instrumentation, and Argo Rollouts canary deploys.
