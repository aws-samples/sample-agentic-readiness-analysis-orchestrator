Detailed Findings

INF-Q1: No managed compute infrastructure HIGH
Category: Infrastructure & DevOps
Finding
Zappa is a CLI tool published to PyPI with no deployed compute infrastructure of its own — no Lambda functions, ECS tasks, or EC2 instances run this tool as a service. Execution happens on developer machines and GitHub-hosted CI runners. No managed compute surface to modernize in-place.
Recommendation
If a hosted deployment service is desired in the future, ECS/Fargate or Lambda are appropriate targets. For the current CLI tool use case, this gap is structural.
INF-Q5: No network security configuration HIGH
Category: Infrastructure & DevOps
Finding
No VPC, subnet, security group, or network segmentation configuration exists. Zappa is a CLI tool executed on developer machines; it has no deployed network infrastructure.
Recommendation
Not applicable for a CLI tool. If Zappa were hosted as a service, private subnets with least-privilege security groups and VPC endpoints would be required.
INF-Q10: No IaC for own infrastructure HIGH
Category: Infrastructure & DevOps
Finding
No IaC defines Zappa’s own operational infrastructure. GitHub Actions workflows exist but no Terraform, CDK, or CloudFormation manages repository settings, secrets, or branch protections. 0% IaC coverage for the project’s own infrastructure.
Recommendation
Consider the Terraform GitHub provider to codify repository settings, branch protection rules, and environment/secret management for auditability and reproducibility.
APP-Q2: Tightly-coupled monolith HIGH
Category: Application Architecture
Finding
Single Python package with no module boundaries. core.py (3,866 LOC) is a god class handling Lambda, API Gateway, S3, CloudFormation, IAM, VPC, and certificate management. cli.py (3,655 LOC) directly instantiates and orchestrates core.py with pervasive cross-module coupling.
Recommendation
Refactor into bounded modules: extract the Lambda handler runtime (handler.py + wsgi.py + asgi.py) as an independent package; split core.py into focused modules (lambda_ops, apigateway_ops, cloudformation_ops, s3_ops); define clean interfaces between CLI and orchestration layers.
