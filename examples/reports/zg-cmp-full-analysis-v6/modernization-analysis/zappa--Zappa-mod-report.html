Detailed Findings

HIGH INF-Q1: No managed compute infrastructure

Category: Infrastructure & DevOps

Finding
No compute infrastructure is defined in the repository. The project is a CLI deployment tool that packages and deploys other applications to AWS Lambda. It does not itself run as a deployed service.
Recommendation
If operational hosting is desired (e.g., Zappa-as-a-Service), consider deploying as a containerized service on EKS. For current CLI usage, this gap is inherent to the tool’s nature.

HIGH INF-Q5: Overly permissive default IAM policies

Category: Infrastructure & DevOps

Finding
Default IAM policies in attach_policy.json use wildcard resources for EC2, Route53, Lambda, X-Ray, S3, DynamoDB, SNS, SQS, and Kinesis operations.
Recommendation
Tighten default IAM policies to use least-privilege resource-scoped permissions. Document network security best practices for users.

HIGH INF-Q10: No Infrastructure as Code

Category: Infrastructure & DevOps

Finding
No IaC files exist in the repository. The tool dynamically generates CloudFormation templates via troposphere for user deployments but its own build/release infrastructure has no IaC definition.
Recommendation
Define release infrastructure as IaC. Consider CDK or Terraform for managing project’s own AWS resources.

HIGH SEC-Q1: No audit logging configuration

Category: Security Baseline

Finding
No CloudTrail or audit logging configuration exists. The tool creates CloudWatch Logs for user Lambda functions but has no audit trail for its own deployment operations.
Recommendation
Add structured audit logging for deployment operations. Document that users should have CloudTrail enabled.

HIGH SEC-Q5: Plaintext secrets in S3 and wildcard IAM permissions

Category: Security Baseline

Finding
Default IAM policies use wildcard permissions. The remote_env feature loads secrets from plaintext JSON in S3 without Secrets Manager integration. No secrets rotation.
Recommendation
Replace wildcard IAM policies with least-privilege. Add Secrets Manager integration for sensitive environment variables.

MEDIUM INF-Q6: No API entry point

Category: Infrastructure & DevOps

Finding
No API entry point exists. The tool is a CLI application with no HTTP/REST surface.
Recommendation
If a service-based deployment API is desired, consider exposing behind API Gateway with authentication.

MEDIUM INF-Q7: No auto-scaling configuration

Category: Infrastructure & DevOps

Finding
No auto-scaling configuration exists. The tool is a CLI application with no persistent service requiring scaling.
Recommendation
Not applicable for current CLI architecture. If converted to a service, implement auto-scaling.

MEDIUM INF-Q11: CI/CD partially automated

Category: Infrastructure & DevOps

Finding
CI is automated via GitHub Actions with multi-version Python testing and linting. CD requires manual workflow_dispatch trigger.
Recommendation
Implement automated semantic release. Add TestPyPI staging before production. Automate version bumping.

MEDIUM APP-Q2: Tightly-coupled monolithic CLI application

Category: Application Architecture

Finding
Single deployable unit with cli.py (3,655 lines) and core.py (3,866 lines) containing mixed concerns.
Recommendation

LOW APP-Q5: No formal API versioning strategy
