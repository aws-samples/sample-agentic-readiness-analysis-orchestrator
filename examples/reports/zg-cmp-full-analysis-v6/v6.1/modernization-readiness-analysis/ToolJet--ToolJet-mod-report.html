Detailed Findings

SEC-Q2: No Encryption at RestHIGH
Category: Security Baseline
Finding

RDS instance has no storage_encrypted parameter configured. No KMS keys defined in any Terraform configuration. All data at rest is unencrypted.

Recommendation

Add storage_encrypted = true with a customer-managed KMS key on RDS. Configure EBS encryption by default. Migrate Redis to ElastiCache with at-rest encryption.

INF-Q7: No Auto-Scaling ConfiguredMEDIUM
Category: Infrastructure & DevOps
Finding

No auto-scaling configured in ECS Terraform (static desired_count = 2). Helm HPA exists but defaults to min=1, max=1 (effectively disabled).

Recommendation

Add ECS Application Auto Scaling with target tracking on CPU/memory and custom metrics. Enable HPA in Helm with appropriate min/max.

INF-Q8: No Backup and Recovery ConfigurationMEDIUM
Category: Infrastructure & DevOps
Finding

RDS instance has no backup_retention_period configured (defaults to 0). No aws_backup_plan resources. skip_final_snapshot = true means no snapshot on deletion.

Recommendation

Configure backup_retention_period = 7 on RDS with PITR enabled. Add aws_backup_plan. Remove skip_final_snapshot = true for production.

INF-Q9: Single-AZ Database Without Fault IsolationMEDIUM
Category: Infrastructure & DevOps
Finding

RDS is single-AZ (no multi_az = true). Redis runs as a sidecar with no replication. An AZ failure takes down the entire data layer.

Recommendation

Enable multi_az = true on RDS. Migrate Redis to ElastiCache with Multi-AZ replication.

OPS-Q2: No SLO DefinitionsMEDIUM
Category: Operations & Observability
Finding

No SLO definitions found. No error budget tracking. No formal definition of acceptable service levels.

Recommendation

Define SLOs for critical user journeys: app loading time (p99 < 3s), API response time (p95 < 500ms), availability (99.9%).

OPS-Q4: No Anomaly Detection or AlertingMEDIUM
Category: Operations & Observability
Finding

No alerting or anomaly detection configured in IaC. No CloudWatch alarms, no PagerDuty/OpsGenie integration.

Recommendation

Add CloudWatch alarms for API error rate, p99 latency, ECS task health. Integrate with SNS for on-call alerting. Enable anomaly detection.

OPS-Q7: No Incident Response AutomationMEDIUM
Category: Operations & Observability
Finding

No runbooks, incident response documentation, or automated remediation. Incident response is entirely ad hoc.

Recommendation

Create runbooks for top failure scenarios. Implement SSM Automation documents for common remediations.

INF-Q2: Partially Managed DatabasesMEDIUM
Category: Infrastructure & DevOps
Finding

Production uses RDS PostgreSQL 16 (managed) but Redis 6.2 runs as self-managed sidecar. RDS is single-AZ.

Recommendation

Migrate Redis to ElastiCache/MemoryDB with Multi-AZ. Enable Multi-AZ on RDS. Consider Aurora PostgreSQL.

INF-Q5: Network Security GapsMEDIUM
Category: Infrastructure & DevOps
Finding

VPC has public subnets only. ECS tasks and RDS in public subnets. No VPC endpoints or PrivateLink.

Recommendation

Restructure VPC with public/private subnet tiers. Place ECS tasks and RDS in private subnets with NAT Gateway.

INF-Q10: Partial IaC CoverageMEDIUM
Category: Infrastructure & DevOps
Finding

Terraform covers compute and networking but not operational resources (alarms, backups, WAF). No shared modules.

Recommendation

Extend IaC to cover CloudWatch alarms, backup plans, WAF rules. Add default_tags to AWS provider.

INF-Q11: Manual DeploymentMEDIUM
Category: Infrastructure & DevOps
Finding

Build automated via GitHub Actions but deployment requires manual trigger. No IaC pipeline. No automated rollback.

Recommendation

Implement continuous deployment pipeline with automated rollback. Add Terraform plan/apply in CI.

APP-Q2: Modular Monolith with Shared DatabaseMEDIUM
Category: Application Architecture
Finding

58 NestJS modules in a single deployable unit. Shared PostgreSQL with cross-module entity access.

Recommendation

Begin Strangler Fig decomposition starting with AI and data-queries modules. Introduce per-module schemas.

SEC-Q5: Secrets in Env Vars Without RotationMEDIUM
Category: Security Baseline
Finding

Credentials in environment variables without rotation. ECS has Secrets Manager IAM but no explicit secret resources.

Recommendation

Define aws_secretsmanager_secret resources. Configure automatic rotation. Reference secrets in ECS task definition.
