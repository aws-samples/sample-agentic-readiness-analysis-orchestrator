Detailed Findings

AUTH-Q5: Credential Management — No Rotation ConfiguredMedium
Finding

Credentials are stored in AWS SSM Parameter Store (SecureString). No credential rotation is configured — SSM Parameter Store does not provide automatic rotation like AWS Secrets Manager.

Gap

No automatic credential rotation mechanism. No evidence of rotation schedules or Lambda-based rotation functions.

Recommendation

Migrate sensitive credentials from SSM Parameter Store to AWS Secrets Manager with automatic rotation enabled.

Effort: MediumFile: terraform/workshop/main.tfLines: 168-180
AUTH-Q6: Immutable Audit Logging — Not ConfiguredMedium
Finding

No CloudTrail configuration found in any Terraform module. No immutable log storage (S3 with Object Lock). No CloudWatch Logs retention policies for audit purposes.

Gap

No audit trail infrastructure defined. Cannot attribute actions to specific principals in an immutable, tamper-evident log store.

Recommendation

Add aws_cloudtrail resource with S3 bucket (Object Lock enabled), CloudTrail log file validation, and multi-region trail.

Effort: MediumFile: terraform/ (all modules)
AUTH-Q7: Agent Identity Suspension — No Immediate MechanismMedium
Finding

IAM roles are defined in Terraform (IRSA). No mechanism for immediate identity suspension without a full Terraform plan/apply cycle. No automated anomaly-triggered suspension.

Gap

No immediate suspension mechanism for agent identities. Revoking an IRSA role requires modifying Terraform state and re-applying.

Recommendation

Implement automation (Lambda + EventBridge) to attach a deny-all inline policy to any IRSA role upon anomaly detection.

Effort: MediumFile: terraform/modules/gitops-saas-infra/main.tfLines: 220-240
