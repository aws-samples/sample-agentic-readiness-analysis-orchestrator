Detailed Findings

AUTH-Q1: No Machine Identity Authentication HIGH

Finding

API Gateway endpoints have no authorization configured. All three REST APIs are publicly accessible without authentication.

Gap

No machine identity mechanism. Cannot attribute API calls to any principal.

Recommendation

Implement API Gateway authorization (IAM SigV4, Cognito, or API keys) on all methods.

Effort: Medium | File: lib/apigateway.ts | Lines: 1-96

AUTH-Q2: No Scoped Permissions MEDIUM

Finding

All Lambda functions receive grantReadWriteData. No differentiated access levels for callers.

Gap

Cannot create read-only agent identity.

Recommendation

Implement per-method IAM authorization that differentiates read vs write access.

Effort: Medium | File: lib/microservice.ts | Lines: 42-88

AUTH-Q3: No Action-Level Authorization MEDIUM

Finding

No action-level authorization. Any caller can invoke any HTTP method on any endpoint.

Gap

Cannot enforce read-only access for agents within a resource type.

Recommendation

Implement IAM authorization with per-method policy conditions.
