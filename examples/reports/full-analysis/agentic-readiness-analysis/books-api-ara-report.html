Detailed Findings

AUTH-Q1: No Machine Identity AuthenticationHigh
Finding

The application uses Amazon Cognito for authentication on POST /books with implicit OAuth flow and USER_PASSWORD_AUTH only. GET /books requires no authentication. No client_credentials flow, API keys, or mTLS configured.

Gap

No machine identity authentication mechanism exists. Cognito is configured for human user authentication only.

Recommendation

Add a Cognito App Client with client_credentials grant type and configure a resource server with custom scopes (e.g., books/read, books/write).
