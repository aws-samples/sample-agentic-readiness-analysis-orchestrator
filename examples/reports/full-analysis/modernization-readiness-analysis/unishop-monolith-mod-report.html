Detailed Findings

INF-Q1: Managed ComputeHIGH
Category: Infrastructure & DevOps
Finding

All compute runs on raw EC2. Application produces executable JAR with embedded launch script deployed directly to EC2. No ECS/EKS, no Lambda, no Fargate.

Recommendation

Containerize and deploy on Amazon EKS with managed node groups. Create Dockerfile, push to ECR, deploy via Helm.

INF-Q2: Managed DatabasesHIGH
Category: Infrastructure & DevOps
Finding

MySQL database likely self-managed on EC2. No IaC defines managed database service. No RDS/Aurora evidence.

Recommendation

Migrate to Amazon Aurora MySQL with automated failover, backups, and Multi-AZ. Use AWS DMS for minimal-downtime migration.

INF-Q5: Network SecurityHIGH
Category: Infrastructure & DevOps
Finding

No VPC, subnet, security group, or network segmentation configuration. Application exposes port 8080 directly.

Recommendation

Define VPC in Terraform: private subnets, least-privilege security groups, VPC endpoints for AWS service access.

INF-Q10: Infrastructure as CodeHIGH
Category: Infrastructure & DevOps
Finding

0% IaC coverage. No Terraform, CloudFormation, CDK, or any IaC. All infrastructure manually provisioned.

Recommendation

Define all infrastructure in Terraform: VPC, EKS, Aurora MySQL, API Gateway, ECR. Use modules for reusability.

INF-Q11: CI/CD AutomationHIGH
Category: Infrastructure & DevOps
Finding

No CI/CD pipeline. No GitHub Actions, Jenkins, CodePipeline, or automation. All deployments manual.

Recommendation

Implement pipeline: lint → build → test → container image → ECR → ArgoCD deploy. Add Terraform plan/apply stages.

SEC-Q5: Secrets ManagementHIGH
Category: Security Baseline
Finding

Plaintext database credentials in application.properties: spring.datasource.password: MonoToMicroPassword. Committed to source control.

Recommendation

Move to AWS Secrets Manager with automated rotation. Remove from application.properties and git history immediately.

SEC-Q2: Encryption at RestHIGH
Category: Security Baseline
Finding

No encryption at rest configured. User PII (email, names) stored unencrypted in MySQL database.

Recommendation

Enable encryption at rest with KMS customer-managed key after Aurora migration. Define key rotation policy.

DATA-Q1: Unstructured Data StorageHIGH
Category: Data Platform
Finding

Product images referenced by name only. S3 SDK in dependencies but not used. No managed object storage configured.

Recommendation

Store product images in S3 with lifecycle policies. Implement document processing with Textract for future needs.

DATA-Q3: Database Engine VersionHIGH
Category: Data Platform
Finding

No database engine version pinned in IaC. MySQL connector 8.0.11 (2018) suggests older server. No lifecycle management.

Recommendation

After Aurora migration, pin engine version in Terraform. Enable minor auto-upgrade. Document major version upgrade procedure.

OPS-Q5: Deployment StrategyHIGH
Category: Operations & Observability
Finding

No deployment strategy. Direct manual JAR deployment to EC2. No blue/green, canary, or rolling updates.

Recommendation

Implement progressive delivery via ArgoCD Rollouts with canary strategy on EKS. Automated rollback on error rate increase.

OPS-Q6: Integration TestingHIGH
Category: Operations & Observability
Finding

Zero automated tests. No src/test/ directory despite spring-boot-starter-test being declared. No integration, unit, or contract tests.

Recommendation

Establish integration tests before decomposition. Use Spring Boot Test with Testcontainers. Add Pact contract tests for API consumers.

APP-Q2: Monolith vs MicroservicesMEDIUM
Category: Application Architecture
Finding

Single Spring Boot JAR with all domains (catalog, basket, users) sharing one database schema. Layered architecture but tight data coupling.

Recommendation

Apply Strangler Fig pattern. Containerize first, then extract basket/order service as first independent microservice.

APP-Q1: Programming LanguagesMEDIUM
Category: Application Architecture
Finding

Java 8 (EOL) + Spring Boot 2.1.x (EOL) + AWS SDK v1 (deprecated). Compound legacy across all three axes.

Recommendation

Upgrade to Java 17+ with Spring Boot 3.x and AWS SDK v2. Prerequisite for modern containerization.
