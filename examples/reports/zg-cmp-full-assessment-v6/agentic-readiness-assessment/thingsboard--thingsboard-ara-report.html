All Findings

HIGHAUTH-Q1: No dedicated machine identity authentication mechanism

Gap: No dedicated machine identity mechanism distinguishable from human users in audit logs. PATs inherit the human user's identity and permissions.

Recommendation: Create a dedicated service account user type with a machine-identity flag that appears in audit log entries.

Priority: P0 | Effort: Medium | Phase: 1

MEDIUMAUTH-Q2: Coarse-grained permission model lacks per-identity scoping

Gap: No fine-grained permission scoping below the three fixed authority levels.

Recommendation: Introduce per-resource-type and per-operation permission grants for agent identities.

Priority: P1 | Effort: High | Phase: 1

MEDIUMAUTH-Q3: Action-level authorization not configurable per identity

Gap: Permissions hardcoded per authority level — not configurable per user or agent identity.

Recommendation: Make permission matrix configurable per user/service account.

Priority: P1 | Effort: High | Phase: 1

MEDIUMAUTH-Q6: Audit logs not stored in immutable storage

Gap: Audit logs are mutable — standard PostgreSQL table without write-once guarantees.

Recommendation: Configure immutable storage (Elasticsearch write-once ILM or S3 Object Lock).

Priority: P1 | Effort: Medium | Phase: 1

MEDIUMAUTH-Q7: No independent agent identity suspension

Gap: No rapid, independent PAT revocation API separate from user account management.

Recommendation: Expose a first-class PAT revocation API endpoint.

Priority: P1 | Effort: Low | Phase: 1

MEDIUMSTATE-Q5: Rate limiting gaps (SYS_ADMIN exempt, no per-identity limits)

Gap: SYS_ADMIN exemption; no per-agent-identity rate limiting.

Recommendation: Add per-identity rate limiting. Remove SYS_ADMIN exemption.

Priority: P1 | Effort: Medium | Phase: 1

MEDIUMDATA-Q1: Sensitive data handling relies on service-layer discipline

Gap: UserCredentials.password lacks @JsonIgnore; no API scope differentiation by sensitivity.

Recommendation: Add @JsonIgnore to password field. Implement OAuth scopes for sensitive data separation.

Priority: P0 | Effort: Medium | Phase: 1

MEDIUMDATA-Q6: No PII redaction in application logs

Gap: No log scrubbing or PII masking middleware.

Recommendation: Add custom logback encoder that masks PII patterns.

Priority: P1 | Effort: Medium | Phase: 1

MEDIUMAPI-Q2: No static OpenAPI specification in repository

Gap: Spec only available from running instance.

Recommendation: Add CI step to generate and commit static OpenAPI spec.

Priority: P2 | Effort: Low | Phase: 2

MEDIUMAPI-Q3: Error responses lack retryable indicator

Gap: No retryable boolean in error responses.

Recommendation: Add retryable field to ThingsboardErrorResponse.

Priority: P2 | Effort: Low | Phase: 2

MEDIUMDISC-Q1: Management API unversioned, no breaking change detection

Gap: No API versioning or contract testing.

Recommendation: Introduce version prefixes and OpenAPI diff in CI.

Priority: P2 | Effort: High | Phase: 2

MEDIUMOBS-Q1: No distributed tracing or structured logging

Gap: No OpenTelemetry, no correlation IDs, plain-text logs.

Recommendation: Integrate OpenTelemetry. Add MDC context to logs.

Priority: P2 | Effort: Medium | Phase: 2

MEDIUMOBS-Q2: No automated alerting rules

Gap: Metrics collected but no alerting for error rates or latency.

Recommendation: Define Prometheus alerting rules.

Priority: P2 | Effort: Low | Phase: 2

MEDIUMENG-Q1: No Infrastructure as Code

Gap: Deployment via Docker Compose only. No IaC governance.

Recommendation: Define infrastructure in declarative IaC.

Priority: P2 | Effort: High | Phase: 2

MEDIUMENG-Q2: CI/CD lacks automated testing

MEDIUMENG-Q3: No automated rollback mechanism
