Detailed Findings

INF-Q1: No managed compute infrastructure HIGH
Category: Infrastructure & DevOps
Finding
Zappa is a CLI tool published to PyPI with no deployed compute infrastructure. No Lambda functions, ECS tasks, or EC2 instances run this tool as a service.
Recommendation
If a hosted deployment service is desired, consider ECS/Fargate or Lambda. For the current CLI use case, this gap is structural.
INF-Q5: No network security configuration HIGH
Category: Infrastructure & DevOps
Finding
No VPC, subnet, security group, or network segmentation. CLI tool with no deployed network infrastructure.
Recommendation
Not applicable for a CLI tool. If deployed as a hosted service, private subnets with least-privilege security groups would be required.
APP-Q2: Tightly-coupled monolith HIGH
Category: Application Architecture
Finding
Single Python package with no module boundaries. core.py (3,866 LOC) is a god class. cli.py (3,655 LOC) directly instantiates and orchestrates core.py with pervasive coupling.
Recommendation
Refactor into bounded modules: separate Lambda handler runtime as independent package, extract AWS service adapters from core.py, define clean interfaces between CLI and orchestration layers.
SEC-Q7: No application security pipeline MEDIUM
Category: Security Baseline
Finding
No SAST, DAST, or dependency vulnerability scanning in CI/CD. Pipeline has lint checks but no security-focused scanning. No Dependabot, pip-audit, safety, Bandit, or Semgrep.
Recommendation
Add Dependabot for dependency updates. Integrate pip-audit for vulnerability detection. Add Bandit or Semgrep for Python SAST. Configure blocking gates on critical findings.
INF-Q10: No IaC for own infrastructure HIGH
Category: Infrastructure & DevOps
Finding
No IaC defines Zappa's own operational infrastructure. GitHub Actions workflows exist but no Terraform/CDK manages repository settings, secrets, or branch protections.
Recommendation
Consider Terraform GitHub provider to codify repository settings, branch protection rules, and environment/secret management.
SEC-Q5: Credentials in environment variables without rotation MEDIUM
Category: Security Baseline
Finding
No plaintext credentials in source. AWS credentials via boto3 credential chain and OIDC for PyPI. However, no Secrets Manager integration and no rotation configured.
Recommendation
Document credential rotation best practices. Consider supporting AWS Secrets Manager for secret retrieval. Encourage IAM roles and SSO over long-lived access keys.
APP-Q6: Environment variables for endpoints only MEDIUM
Category: Application Architecture
Finding
Uses environment variables for AWS endpoint configuration via boto3. No dynamic service discovery, but boto3 endpoint resolution is adequate for the use case.
Recommendation
No action needed. boto3's endpoint resolution is the correct pattern for AWS SDK-based tools.
DATA-Q1: S3 usage without parsing pipeline LOW
Category: Data Platform

Zappa uses S3 for deployment package storage (ZIP uploads) but has no parsing or extraction pipeline for the stored data.
