Detailed Findings

HIGH INF-Q5: No Network Security Infrastructure

Category: Infrastructure & DevOps

Finding

No IaC defines VPC, subnets, security groups, NACLs, or any network segmentation. Docker Compose examples expose services on default Docker networking with no isolation.

Recommendation

Define VPC with private subnets for databases and public subnet for the Zipkin server. Create security groups with least-privilege rules. Use VPC endpoints for AWS service access.

HIGH INF-Q10: Zero Infrastructure as Code Coverage

Category: Infrastructure & DevOps

Finding

No Terraform, CloudFormation, CDK, Helm charts, or Kustomize files exist. All infrastructure would need to be created manually.

Recommendation

Create Terraform modules defining EKS cluster, VPC/networking, Aurora MySQL, OpenSearch domain, MSK cluster, ALB, and monitoring.

HIGH SEC-Q1: No Audit Logging

Category: Security Baseline

Finding

No CloudTrail or equivalent audit logging configuration exists. No forensic trail for API access or infrastructure changes.

Recommendation

Define CloudTrail in IaC with log file validation and S3 Object Lock for immutable storage.

HIGH SEC-Q2: No Encryption at Rest

Category: Security Baseline

Finding

No KMS or encryption at rest configuration exists. Docker-based databases use default unencrypted storage.

Recommendation

Enable Aurora storage encryption with KMS, OpenSearch domain encryption, and define customer-managed KMS keys.

HIGH OPS-Q2: No SLO Definitions

Category: Operations & Observability

Finding

No SLO definitions exist. No CloudWatch alarms on latency percentiles or error budgets. Micrometer metrics available but not consumed by SLO definitions.

Recommendation

Define SLOs for query API p99 latency, span ingestion success rate, and service availability. Implement error budget tracking.
