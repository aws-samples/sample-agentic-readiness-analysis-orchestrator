Detailed Findings

HIGH SEC-Q1: No CloudTrail Audit Logging

Category: Security Baseline
Finding
No CloudTrail configuration found in any Terraform module. Application-level audit logs exist but no AWS infrastructure audit trail.
Recommendation
Add aws_cloudtrail resource to Terraform with log file validation and S3 Object Lock for immutable storage.

HIGH OPS-Q2: No SLO Definitions

Category: Operations & Observability
Finding
No SLO definitions found. Custom metrics exist but not tied to SLO targets. No error budget tracking.
Recommendation
Define SLOs for critical user journeys: app builder load time, data query success rate, workflow execution success rate.

MEDIUM INF-Q7: No Auto-Scaling Configured

Category: Infrastructure & DevOps
Finding
No auto-scaling configured. ECS uses static desired_count=2. Kubernetes HPA disabled. No scaling policies on any resource.
Recommendation
Configure ECS Application Auto Scaling with target tracking. For EKS, configure HPA with custom metrics.

MEDIUM INF-Q8: No Backup and Recovery

Category: Infrastructure & DevOps
Finding
RDS has no backup_retention_period (defaults to 0). skip_final_snapshot=true. No AWS Backup plan. Redis data is ephemeral.
Recommendation
Set backup_retention_period=7 on RDS with PITR. Create AWS Backup plan. Migrate Redis to ElastiCache with backup.

MEDIUM OPS-Q3: No Business Metrics

Category: Operations & Observability
Finding
Only infrastructure metrics (api.hits, api.duration, users.concurrent). No business outcome metrics tracked.
Recommendation
Add business metrics: apps_created_total, data_queries_success/failure, workflows_completed/failed.

MEDIUM OPS-Q4: No Anomaly Detection or Alerting

Category: Operations & Observability
Finding
No CloudWatch alarms in Terraform. No alerting beyond Sentry error reporting. No anomaly detection.
Recommendation
Add CloudWatch alarms for API error rate, p99 latency, ECS task failures. Enable anomaly detection on critical paths.

MEDIUM OPS-Q7: No Incident Response Automation

Category: Operations & Observability
Finding
No runbooks, SSM Automation documents, or self-healing automation. Incident response is entirely ad hoc.
Recommendation
Create runbooks for common incidents. Implement as SSM Automation documents. Add self-healing patterns.

MEDIUM INF-Q2: Self-Managed Redis and Single-AZ RDS

Category: Infrastructure & DevOps
Finding
RDS managed but single-AZ. Redis runs as self-managed sidecar container.
Recommendation
Enable Multi-AZ on RDS. Migrate Redis to ElastiCache or MemoryDB with Multi-AZ.

MEDIUM INF-Q5: Public Subnets with Missing Segmentation

Category: Infrastructure & DevOps
Finding
VPC with public subnets only. No private subnets for ECS tasks and RDS. Missing VPC endpoints.
Recommendation
Restructure VPC with private subnets. Add NAT gateway and VPC endpoints for AWS services.

MEDIUM INF-Q9: Database Single-AZ Deployment

Category: Infrastructure & DevOps
Finding
Compute spans 2 AZs but RDS is single-AZ. AZ failure would take down database.
Recommendation
Enable multi_az=true on RDS. Migrate Redis to ElastiCache with Multi-AZ failover.

MEDIUM INF-Q10: Partial IaC Coverage

Category: Infrastructure & DevOps
Finding
Terraform covers compute/VPC/ALB/RDS but NOT monitoring, alerting, backup, auto-scaling, or WAF.
Recommendation
Expand Terraform to cover CloudWatch alarms, AWS Backup plans, auto-scaling policies, WAF rules.

MEDIUM APP-Q2: Monolith with Shared Database

Category: Application Architecture
Finding
Single NestJS server with 50+ modules sharing PostgreSQL via TypeORM. Identifiable module boundaries but shared schemas.
Recommendation
Begin Strangler Fig decomposition starting with Workflows, AI, and ToolJet Database modules.

MEDIUM APP-Q5: API Versioning Not Used

Category: Application Architecture
Finding
NestJS versioning framework configured but defaults to VERSION_NEUTRAL. No active version paths.
Recommendation
Apply @Version decorators as APIs evolve. Version public API surface first.

MEDIUM DATA-Q1: Unstructured Data Not in S3

Category: Data Platform
Finding
Files stored on local filesystem/container volume. No S3 for application storage.
Recommendation
Migrate file storage to Amazon S3 with presigned URLs and lifecycle policies.

MEDIUM SEC-Q2: No Infrastructure Encryption at Rest

Category: Security Baseline
Finding
No KMS keys in Terraform. RDS lacks explicit storage_encrypted. App-level AES-256-GCM exists but no infra-level encryption.
Recommendation
Add storage_encrypted=true and customer-managed KMS key to RDS and all data stores.

MEDIUM SEC-Q5: Credentials in Terraform Defaults

Category: Security Baseline
Finding
Terraform variables have default credentials (PG_PASS="postgres"). No rotation configured.
Recommendation
Remove defaults from sensitive variables. Implement Secrets Manager rotation.

MEDIUM SEC-Q7: No SAST in CI Pipeline

Category: Security Baseline
Finding
Dependency scanning active but no SAST for code vulnerabilities. No container image scanning.
Recommendation
Add SAST scanning and container scanning to CI/CD pipeline.

MEDIUM OPS-Q5: Rolling Deployments Only

Category: Operations & Observability
Finding
Kubernetes RollingUpdate with health checks. No canary or blue/green deployment.
Recommendation
Implement canary deployments with Argo Rollouts or CodeDeploy blue/green.

MEDIUM OPS-Q6: Integration Tests Not in CI

Category: Operations & Observability
Finding
Cypress tests exist but require manual label triggers. Not run on every PR.

MEDIUM OPS-Q8: No Observability Ownership

Category: Operations & Observability
Finding
CODEOWNERS covers only package.json and migrations. No dashboards, alarm ownership, or team attribution.
Recommendation
Extend CODEOWNERS for observability. Create per-module dashboards with named owners.

MEDIUM OPS-Q9: Inconsistent Resource Tagging

Category: Operations & Observability
Finding
Minimal tagging. No default_tags in Terraform. No cost allocation or environment tags.

Add default_tags to Terraform AWS provider. Implement required-tags Config rules.
